Why I love Linux

Hi! My name is Kasper Andersson and I'm very interested in operating systems, development and the like. One of my favorite projects is Linux, a free and open source operating system kernel, often used with software from Android or the GNU project. A kernel can be seen as the heart of the operating system. It's what allows the applications on your computer to interact with the hardware and it allocates processing power and memory so everything runs smoothly.

What can you use Linux for?

Pretty much anything you'd use any other OS for. I personally use it for a lot of my school work, producing music, gaming, development and everything else. It's traditionally been run mostly on servers due to the lack of desktop software, like games and Photoshop, but in recent years projects like Wine and Proton have made gaming on Linux viable to the point where Valve, the company behind Steam, is now releasing a game console which runs Linux and is able to play almost all Windows games.

It's also a very usable and stable OS, I run it on everything from my gaming PC to my laptop to my server, which hosts a live version of this page, a Git page and a Nextcloud instance.

Why use Linux?

Sometimes I get asked why I use Linux and not a more mainstream OS, like Windows or macOS. There are many reasons to use it but here are just a few:

Simple updates

If you use one of the more mainstream versions of Linux, like Ubuntu , Mint or Fedora , updates are super simple, not forced on you (though you really should install security updates), and most don't even need you to restart your computer. You can even tell it to update in the background and the updates will be automatically downloaded in the background and applied the next time you restart you computer.

This comes as a result of Linux traditionally being a server operating system, where you want your servers to be shut off for as short a time as possible.

Licensing

Linux is free and open source software, which means you are allowed to use the code for pretty much anything you want, you can study it, change it, and publish your changes for others to use, as long as you let others do the same with your changes.

I really like this model of development because it allows the people who want a function in their software to help develop it. You also don't get the problems of planned absolecense that other software faces, like phones not receiving updates to encourage you to buy a new one.

You own your computer

Tying into the previous point, using Linux you are the one who controls your operating system. The companies who make Linux and derivative operating systems don't make money off selling the operating system itself. They let anyone use their version and publish it in the hope that companies will pay for support for their servers. Because of this, they don't have any real incentive to collect more data than needed (which often is none) or prevent you from using certain features.
